Analyte EFO Mapper

Maps proteins/metabolites to measurement terms and can also map disease/phenotype reported traits using a curated cache plus EFO/OBO fallback.

1. Setup

If you already have Python and this repo cloned, this is the shortest setup + run path.

First run on a new machine: execute setup-bundled-caches once to validate bundled protein/metabolite/trait caches and local index.
  1. Edit data/analytes.tsv with your queries.
  2. Run the command below.
  3. Use analytes_efo.tsv as main output and withheld_for_review.tsv for manual follow-up.
.venv/bin/python skills/pqtl-measurement-mapper/scripts/map_measurement_efo.py map \
  --input data/analytes.tsv \
  --output final_output/analytes_efo.tsv \
  --review-output final_output/withheld_for_review.tsv \
  --index skills/pqtl-measurement-mapper/references/measurement_index.json \
  --entity-type auto \
  --workers 8 \
  --parallel-mode process \
  --progress

Setup Details (Novice)

Do this once on a new machine. Follow the exact steps below even if you have never used Python before.

  1. Open your command line: use Terminal (Mac/Linux) or PowerShell (Windows).
  2. Install Git and Python 3.10+: if missing, install from git-scm.com/downloads and python.org/downloads. Then check versions: git --version, python3 --version (Mac/Linux), or py --version (Windows).
  3. Copy one full setup block below and run it: this downloads the project, creates an isolated Python environment, installs required packages, creates folders, and copies your input template to data/analytes.tsv.
If a command says not found, stop there, install the missing tool (Git or Python), then run the block again.

Mac/Linux: run this exactly

git clone https://github.com/earlEBI/analyte-efo-mapper.git
cd analyte-efo-mapper
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -U pip
python -m pip install -r requirements.txt
mkdir -p data final_output
cp docs/input_template.tsv data/analytes.tsv
.venv/bin/python skills/pqtl-measurement-mapper/scripts/map_measurement_efo.py setup-bundled-caches

Windows PowerShell: run this exactly

git clone https://github.com/earlEBI/analyte-efo-mapper.git
cd analyte-efo-mapper
py -3 -m venv .venv
.venv\Scripts\Activate.ps1
python -m pip install -U pip
python -m pip install -r requirements.txt
New-Item -ItemType Directory -Force data, final_output | Out-Null
Copy-Item docs\input_template.tsv data\analytes.tsv
.venv\Scripts\python.exe skills/pqtl-measurement-mapper/scripts/map_measurement_efo.py setup-bundled-caches
Next: edit data/analytes.tsv with your own IDs (UniProt, gene symbol, gene ID, protein name, or metabolite IDs/names).
The offline cache setup command validates bundled protein/metabolite/trait caches and builds the local index without internet downloads.

Prepare your input file before running the copy-paste commands below.

4. Additional Options

Use these only when you need context tuning, cache refresh, or deeper review outputs.

Optional HMDB Metabolite Cache (Recommended)

For reproducible metabolite mapping, use your pinned HMDB file version and build the cache once.

Normal mapping does not require a local HMDB file. The shipped metabolite_aliases.tsv is used by default. 
mkdir -p skills/pqtl-measurement-mapper/references/hmdb_source
cp /path/to/your/structures.sdf skills/pqtl-measurement-mapper/references/hmdb_source/structures.sdf
.venv/bin/python skills/pqtl-measurement-mapper/scripts/map_measurement_efo.py metabolite-alias-build \
  --output skills/pqtl-measurement-mapper/references/metabolite_aliases.tsv \
  --no-merge-existing
.venv/bin/python skills/pqtl-measurement-mapper/scripts/map_measurement_efo.py metabolite-alias-build \
  --output skills/pqtl-measurement-mapper/references/metabolite_aliases.tsv \
  --hmdb-xml /path/to/structures.sdf \
  --no-merge-existing
By default, the command auto-detects HMDB files from references/hmdb_source and references/metabolite_downloads.
If no local file is found, it auto-downloads your pinned HMDB bundle URL (--hmdb-url, default points to this repo's GitHub release asset).
Maintainer step: publish structures.sdf.gz as a GitHub Release asset so all users pull the same HMDB version.
If users do rebuild and have no local HMDB file, the command fetches your pinned bundle from GitHub release automatically.
For fully offline installs, include structures.sdf or structures.sdf.gz inside references/hmdb_source/. 
.venv/bin/python skills/pqtl-measurement-mapper/scripts/map_measurement_efo.py metabolite-alias-build \
  --output skills/pqtl-measurement-mapper/references/metabolite_aliases.tsv \

Measurement Context and Optional Arguments

Use --measurement-context to control tissue/matrix filtering and --additional-contexts to opt into extras.

  • --measurement-context blood (default): allows blood + plasma + unlabeled terms, excludes serum.
  • --measurement-context plasma: plasma-only context.
  • --measurement-context blood --additional-contexts serum: blood/plasma plus serum.
  • --measurement-context cerebrospinal_fluid: CSF-only context.
  • --additional-context-keywords <phrase>: free-text add-on (for example aorta, adipose tissue).
.venv/bin/python skills/pqtl-measurement-mapper/scripts/map_measurement_efo.py map \
  --input data/analytes.tsv \
  --output final_output/analytes_efo.tsv \
  --index skills/pqtl-measurement-mapper/references/measurement_index.json \
  --measurement-context plasma \
  --workers 8 \
  --parallel-mode process \
  --progress
.venv/bin/python skills/pqtl-measurement-mapper/scripts/map_measurement_efo.py map \
  --input data/analytes.tsv \
  --output final_output/analytes_efo.tsv \
  --index skills/pqtl-measurement-mapper/references/measurement_index.json \
  --measurement-context auto \
  --additional-context-keywords aorta \
  --workers 8 \
  --parallel-mode process \
  --progress
.venv/bin/python skills/pqtl-measurement-mapper/scripts/map_measurement_efo.py map \
  --input data/analytes.tsv \
  --output final_output/analytes_efo.tsv \
  --review-queue-output final_output/review_queue.tsv \
  --index skills/pqtl-measurement-mapper/references/measurement_index.json \
  --entity-type auto \
  --workers 8 \
  --parallel-mode process \
  --progress
  • Useful optional flags: --top-k, --min-score, --workers, --name-mode, --entity-type, --auto-enrich-uniprot, --metabolite-aliases, --unmapped-output, --review-output, --review-queue-output, --withheld-triage-output.
--auto-enrich-uniprot is incremental: it fetches missing aliases only for accession-like queries in your current input, writes them to uniprot_aliases.tsv, then rebuilds the index. It is not a full UniProt rebuild.

Review & Maintenance

Use these outputs and advanced commands for QC, triage, and cache refresh.

Outputs

  • final_output/analytes_efo.tsv: main mappings with confidence and validation.
  • final_output/withheld_for_review.tsv (optional): only candidates withheld from auto-validation.
  • final_output/review_queue.tsv (optional): broader manual-review file with closest unresolved candidates; useful suggestions, but many are likely incorrect.
  • final_output/withheld_for_review_triage.tsv (optional): top withheld candidates triaged to reject_high, review_needed, accept_medium, accept_high, with query_primary_label and suggested_subject for full-name QC.
  • final_output/analytes_unmapped.tsv: optional pure unresolved list.
  • Ontology IDs are written with underscores (for example EFO_0802947).
This website is documentation and templates. Mapping runs in your local Python environment.

Advanced: Withheld Triage (Optional)

Use this when you want to prioritize withheld candidates into likely accept/reject buckets for faster manual review.

.venv/bin/python skills/pqtl-measurement-mapper/scripts/map_measurement_efo.py map \
  --input data/analytes.tsv \
  --output final_output/analytes_efo.tsv \
  --review-output final_output/withheld_for_review.tsv \
  --withheld-triage-output final_output/withheld_for_review_triage.tsv \
  --index skills/pqtl-measurement-mapper/references/measurement_index.json \
  --entity-type auto \
  --workers 8 \
  --parallel-mode process \
  --progress
Requires --review-output in the same run. This is optional and does not replace human QC.

Refresh EFO Cache (Optional)

Use this when you want to refresh the full EFO/OBA measurement cache and rebuild the local index. Rebuild is on by default and should usually stay on.

.venv/bin/python skills/pqtl-measurement-mapper/scripts/map_measurement_efo.py refresh-efo-cache \
  --download-url https://github.com/EBISPOT/efo/releases/latest/download/efo.obo \
  --term-cache skills/pqtl-measurement-mapper/references/efo_measurement_terms_cache.tsv \
  --index skills/pqtl-measurement-mapper/references/measurement_index.json \
  --analyte-cache skills/pqtl-measurement-mapper/references/analyte_to_efo_cache.tsv \
  --uniprot-aliases skills/pqtl-measurement-mapper/references/uniprot_aliases.tsv \
  --metabolite-aliases skills/pqtl-measurement-mapper/references/metabolite_aliases.tsv
No local OBO file is required. If --efo-obo is not provided, the script downloads from the URL above.

Troubleshooting

  • python3: command not found: install Python and restart terminal.
  • ModuleNotFoundError: activate virtual environment and run python -m pip install -r requirements.txt.
